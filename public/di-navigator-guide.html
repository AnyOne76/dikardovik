В приложение

Кадровый Навигатор

Веб-сервис для кадровой службы: из введённой должности и подразделения автоматически собирается должностная инструкция по вашему корпоративному шаблону — с табличным предпросмотром, историей версий и выгрузкой в Word (DOCX).

Зачем это нужно

HR не копирует текст из десяти разных источников вручную: система сначала подтягивает справочную информацию (в первую очередь с assistentus.ru, при нехватке данных — через Perplexity), затем модель на OpenRouter оформляет материал в строгую структуру документа: разделы «Общие положения», «Должностные обязанности», «Права», «Ответственность», шапка и блоки согласования — как в ваших образцах.

Внутри приложения действуют дополнительные правила (например, единый текст про приём на работу, особые пункты для руководителей или линейных сотрудников, корректировка формулировок). Итог можно править вручную, сохранять новыми версиями и точечно перегенерировать отдельные блоки.

Что умеет пользователь

  • Вход по email и паролю (NextAuth).
  • Выбор подразделения из фиксированного списка или вариант «Другое» со своим названием.
  • Генерация ДИ по должности: предпросмотр в виде таблицы, экспорт в DOCX.
  • Переход в редактор сразу после генерации и из списка истории — правка по секциям, предпросмотр, сохранение новой версии.
  • Перегенерация блока с повторным поиском по должности и подразделению (assistentus + Perplexity) и доработкой через OpenRouter.
  • История сохранённых документов, выборочное удаление, скачивание DOCX.
  • Администратор — управление пользователями, смена логина, сброс пароля (отдельные страницы в приложении).

На каких технологиях сделано

Next.js 16 + React 19

Современный интерфейс и серверные API на одной платформе (App Router).

TypeScript

Предсказуемая типизация кода и контрактов данных.

Prisma + SQLite

Пользователи, запуски генерации, версии ДИ, связи между должностями — в одной файловой базе.

NextAuth + bcrypt

Вход по логину и паролю, хэширование паролей, сессии.

Zod

Проверка JSON должностной инструкции: структура и заголовки секций не могут «поехать».

docx

Формирование файла Word с табличной вёрсткой, согласованной с предпросмотром.

Tailwind CSS 4

Стили экранов приложения (эта справочная страница — отдельный самодостаточный HTML).

Внешние API

Perplexity, OpenRouter; парсинг справки с assistentus.ru на сервере.

Безопасность и данные (кратко)

  • Закрытые маршруты и API проверяют авторизацию; разделение ролей (например, admin и hr).
  • Ограничение частоты запросов на чувствительных операциях (вход, генерация, перегенерация).
  • Секреты и ключи API задаются только в переменных окружения на сервере, не в репозитории.
  • Каждая сгенерированная инструкция хранится как версия с JSON-шаблоном и текстом для поиска/экспорта.

Карта потоков

Ниже — две версии одной упрощённой схемы: слева направо (обзор в одну строку) и по шагам сверху вниз (удобно при просмотре файла целиком).

Редактор читает и сохраняет JSON через /api/di/history/[id]; перегенерация одного блока — отдельный запрос к API с текущим шаблоном и повторным поиском по должности.

Полная схема (слои и маршруты)

Полная схема ниже в одном блоке: четыре уровня (интерфейс, API, внешние сервисы, база данных), внутри каждого уровня — две колонки. Полностью помещается по ширине страницы без прокрутки.

Как читать полную схему: сверху — что видит пользователь; ниже — какие вызовы уходят на сервер; затем — внешние API и этап нормализации/валидации; внизу — что сохраняется в базе. «Упрощённая» схема выше остаётся для быстрого объяснения цепочки в одну линию.

Сохранение копии

Кнопка вверху открывает диалог сохранения. При необходимости отключите в нём колонтитулы (дата, адрес страницы), чтобы в файл попадал только текст документа.
